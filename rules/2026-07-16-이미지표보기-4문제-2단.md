42. [양식 파일] 판단 조건: 양식 파일이 제공되면 → 처리 방법: 양식을 수정하지 않고 항상 다른 이름으로 저장한다 → 최종 출력: 새 HWPX 파일을 만든다 → 예외·검토 필요: 원본 양식 파일을 덮어쓰지 않는다.
43. [양식 파일 없음] 판단 조건: 양식 파일이 없으면 → 처리 방법: 문서를 2단으로 나눈 뒤 전사를 시작한다 → 최종 출력: 2단 문서에 전사 결과를 작성한다 → 예외·검토 필요: 양식이 있으면 이 자동 2단 설정을 양식에 다시 적용하지 않는다.
44. [문서 시작] 판단 조건: HWPX 작성을 시작하면 → 처리 방법: 문서 맨 위로 이동한다 → 최종 출력: 첫 이벤트부터 문서 시작 위치에 작성한다 → 예외·검토 필요: 기존 커서 위치에 이어 쓰지 않는다.
45. [4문제 배치] 판단 조건: 문항을 순서대로 작성하면 → 처리 방법: 1번·2번 작성 후 단 강제 나누기, 3번·4번 작성 후 강제 쪽 나누기를 반복한다 → 최종 출력: 한 쪽에 네 문제씩 배치한다 → 예외·검토 필요: 양식의 기존 구역·쪽 설정은 보존한다.
46. [미주 내용] 판단 조건: `#N`으로 만든 미주가 있으면 → 처리 방법: 미주 내용을 비워 둔다 → 최종 출력: 빈 미주만 삽입한다 → 예외·검토 필요: 빠른 답지·해설은 후속 기능에서 추가한다.
47. [규칙 버전] 판단 조건: 규칙을 추가하거나 수정하면 → 처리 방법: 기존 MD를 덮어쓰지 않고 새 MD 파일로 저장한다 → 최종 출력: 가장 최근에 선택·작성한 규칙을 사용한다 → 예외·검토 필요: 사용자가 별도 규칙을 지정하면 그 규칙을 우선한다.
