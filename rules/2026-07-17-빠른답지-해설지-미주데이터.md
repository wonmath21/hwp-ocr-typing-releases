43. [양식 파일 없음] 판단 조건: 양식 파일이 없으면 → 처리 방법: 문서를 2단으로 나눈 뒤 전사를 시작한다 → 최종 출력: 2단 문서에 전사 결과를 작성한다 → 예외·검토 필요: 양식이 있으면 이 자동 2단 설정을 양식에 다시 적용하지 않는다.
44. [문서 시작] 판단 조건: HWPX 작성을 시작하면 → 처리 방법: 문서 맨 위로 이동한다 → 최종 출력: 첫 이벤트부터 문서 시작 위치에 작성한다 → 예외·검토 필요: 기존 커서 위치에 이어 쓰지 않는다.
45. [4문제 배치] 판단 조건: 문항을 순서대로 작성하면 → 처리 방법: 1번·2번 작성 후 단 강제 나누기, 3번·4번 작성 후 강제 쪽 나누기를 반복한다 → 최종 출력: 한 쪽에 네 문제씩 배치한다 → 예외·검토 필요: 양식의 기존 구역·쪽 설정은 보존한다.
46. [미주 내용] 판단 조건: `#N`으로 만든 미주가 있으면 → 처리 방법: 아래 `빠른답지·해설지 미주 데이터 규칙`에 따라 정확히 연결된 빠른답지·해설지 원문만 해당 미주에 기록한다 → 최종 출력: 일치하는 원본 데이터가 있을 때만 `<미주 N>` 데이터를 사용하고, 일치하는 원본 데이터가 없으면 기존의 빈 HWP 미주를 그대로 둔다 → 예외·검토 필요: 번호가 불명확하거나 문제 본문에 없는 번호는 미주를 만들지 않고 `검수 메모`에 기록한다.
47. [규칙 버전] 판단 조건: 규칙을 추가하거나 수정하면 → 처리 방법: 기존 MD를 덮어쓰지 않고 새 MD 파일로 저장한다 → 최종 출력: 가장 최근에 선택·작성한 규칙을 사용한다 → 예외·검토 필요: 사용자가 별도 규칙을 지정하면 그 규칙을 우선한다.

48. [업데이트 노트] 선택지 세로 배열, 1·3번 계열의 8회 `[enter]`, 지수 중괄호 규칙을 추가한다.
49. [선택지 세로 배열] 판단 조건: 문항에 선택지 ①~⑤가 있으면 → 처리 방법: 각 선택지를 새 줄에 배열한다 → 최종 출력: 각 선택지 ①~⑤는 반드시 직전 `[enter]` 뒤에서 시작한다. 선택지는 존재할 때만 세로 배치하며 문항 끝을 판단하지 않는다. → 예외·검토 필요: 선택지가 없는 문항에는 선택지 블록을 만들지 않는다.
50. [문항 끝 간격] 판단 조건: 다음 상위 문항 표식 `#N`이 존재하고 현재 문항 번호 `n`을 4로 나눈 나머지가 1 또는 3이면 → 처리 방법: 문항 끝은 다음 `#N` 바로 전이며, 다음 `#N`이 없으면 입력의 끝이다. 현재 문항의 마지막 본문·선택지·이미지 표식 뒤, 다음 `#N` 바로 전에 줄바꿈 표식을 여덟 개 기록한다 → 최종 출력: 문항 끝과 다음 `#N` 사이에 `[enter]`를 정확히 8회 둔다. → 예외·검토 필요: 마지막 문항에는 번호와 관계없이 레이아웃용 `[enter]`를 추가하지 않는다.
51. [문항 끝 배치] 판단 조건: 다음 상위 문항 표식 `#N`이 존재하고 현재 문항 번호 `n`을 4로 나눈 나머지가 2 또는 0이면 → 처리 방법: 현재 문항의 마지막 본문·선택지·이미지 표식 뒤, 다음 `#N` 바로 전에 `[enter]`를 정확히 1회 기록하고 기존 변환기의 강제 나누기를 적용한다 → 최종 출력: `n mod 4`가 2 또는 0이면 문항 끝과 다음 `#N` 사이에 `[enter]`를 정확히 1회 둔다. `n mod 4`가 2이면 다음 `#N` 시작 시 강제 단나눔, 0이면 강제 쪽나눔을 적용한다. → 예외·검토 필요: 다음 `#N`이 없는 마지막 문항에는 레이아웃용 `[enter]`와 강제 나누기용 표식을 추가하지 않는다.
52. [지수 중괄호] 판단 조건: `@...@` 안에 지수 연산자 `^`가 있으면 → 처리 방법: 모든 `^` 뒤의 지수 전체를 중괄호로 감싼다 → 최종 출력: `@...@` 안의 모든 `^`는 `^{...}`만 허용한다.
허용: `@2^{2}-1@`
허용: `@(5x+A)^{2}=25x^{2}+Bx+4@`
금지: `@2^2-1@`
금지: `@(5x+A)^2=25x^2+Bx+4@`
금지: `@(5x+A)^{2=25x^{2}+Bx+4}@`
→ 예외·검토 필요: 이전 규칙의 중괄호 없는 지수 예시는 이 규칙으로 대체하며, 중괄호 짝이 맞지 않는 수식은 출력하지 않는다.

53. [문항 경계 예시] 선택지 유무와 관계없이 문항 경계 규칙을 적용한다.

```text
#1선택지가 없는 문항의 마지막 내용[enter][enter][enter][enter][enter][enter][enter][enter]#2다음 문항
#3선택지가 있는 문항[enter]① 보기 1[enter]② 보기 2[enter]③ 보기 3[enter]④ 보기 4[enter]⑤ 보기 5[enter][enter][enter][enter][enter][enter][enter][enter]#4다음 문항
#5마지막 문항의 마지막 내용
```

위 예시에서 `#5` 뒤에는 레이아웃용 `[enter]`를 추가하지 않는다.

54. [빠른답지·해설지 미주 데이터 규칙] 이 절의 규칙은 빠른답지·정답표·해설지·풀이지가 실제 입력에 포함된 경우에만 적용한다. 문제 본문 전사와 답지·해설 전사를 절대로 섞지 않으며, 문제 내용은 `붙여넣기용 타이핑`에만, 빠른답지·해설지 내용은 `미주 데이터`에만 둔다.
55. [미주 번호 연결] 판단 조건: 빠른답지·해설지에 문항 번호가 있고 문제 본문에 같은 `#N`이 있으면 → 처리 방법: 명시적으로 같은 번호인 경우에만 연결한다 → 최종 출력: 해당 번호의 `<미주 N>`만 만든다 → 예외·검토 필요: `1번`, `1.`, `문제 1`, `①`로는 번호를 추론하거나 연결하지 않는다. 번호가 없거나 연결이 불확실하면 배정하지 않고 `검수 메모`에 기록한다. 답지·해설지 번호가 문제 본문에 존재하지 않으면 `<미주 N>`을 출력하지 않고 `검수 메모`에 기록한다. 문제 본문에 문항은 있으나 답지·해설에 그 번호가 없으면 `<미주 N>` 블록을 만들지 않고 기존의 빈 HWP 미주를 그대로 둔다.
56. [빠른답지 전사] 판단 조건: 정확히 연결된 빠른답지 원문이 있으면 → 처리 방법: 원문 정답을 바꾸지 않고 해당 `<미주 N>` 안에서 `정답:`으로 시작해 기록한다 → 최종 출력: `정답: ③`, `정답: @x=2@`, `정답: ①, ④`, `정답: ㄱ, ㄷ`, `정답: @-3@`처럼 원래 표기를 보존한다 → 예외·검토 필요: 수식·숫자·로마자·수학 기호가 포함된 답은 기존 수식 규칙에 따라 `@수식@`으로 기록한다. 점수·난이도·출제 의도·단원명·교사 메모는 원문에서 명백히 정답 일부인 경우 외에는 무시한다.
57. [빠른답지 특수 표기와 불일치] 판단 조건: 빠른답지에 특수 정답 상태가 있거나 빠른답지 정답과 해설 결론이 다르면 → 처리 방법: `복수 정답`, `정답 없음`, `조건부 정답`, `오류`, `출제 제외`를 `정답:` 뒤에 원문 그대로 보존하고, 불일치하면 두 원문을 모두 보존한다 → 최종 출력: 빠른답지 원문은 `정답:`에, 해설 결론은 `해설:`에 각각 남긴다 → 예외·검토 필요: 두 자료의 결론 불일치는 `검수 메모`에 기록하며, 문제를 풀어서 빠진 정답을 만들지 않는다.
58. [해설지 전사] 판단 조건: 정확히 연결된 해설지 원문이 있으면 → 처리 방법: 해당 `<미주 N>` 안에서 `해설:`로 시작해 문장·수식·계산 순서·조건·결론을 원래 순서대로 기록한다 → 최종 출력: 해설 원문을 요약·재구성·보충·수정·재풀이하지 않은 전사 결과를 출력한다 → 예외·검토 필요: 해설 안에 반복된 문제 문장은 완전히 동일하게 반복된 부분만 생략한다. 이해에 필요한 추가 조건·변형 수식·표·도형 설명은 보존한다.
59. [해설 수식·문단·이미지] 판단 조건: 해설에 수식·문단 나눔·필수 그림·표·그래프가 있으면 → 처리 방법: 수식은 기존 `@수식@` 규칙을 적용하고 모든 지수는 `^{...}`로 기록하며, 해설 문단 나눔은 `[enter]`로 기록한다 → 최종 출력: 필요한 그림·표·그래프는 `[image: 파일명.png]`로, 판독할 수 없는 해설 그림·표는 `[확인 필요: 해설 그림·표 판독 불가]`로 기록한다 → 예외·검토 필요: LaTeX 문법을 사용하지 않으며, 이미지 표식에는 파일명만 쓴다.
60. [빠른답지와 해설지 우선순위] 판단 조건: 빠른답지와 해설지가 함께 있거나 해설지만 있으면 → 처리 방법: 빠른답지와 해설지가 함께 있으면 빠른답지의 원래 답을 `정답:`에 사용하고 해설이 명시한 결론은 `해설:`에 유지한다. 해설지만 있으면 `정답:`을 만들지 않는다 → 최종 출력: 둘 다 있으면 `정답:`을 먼저, `해설:`을 다음에 출력하고, 해설지만 있으면 명시적 결론도 `해설:` 안에만 둔다 → 예외·검토 필요: 해설로 정답을 추정하지 않는다.
61. [미주 문단·강제줄나눔] 판단 조건: 하나의 미주에서 빠른답지 뒤에 해설이 이어지고, 해설 원문이 한 문단 안 줄바꿈을 요구하면 → 처리 방법: 답 바로 뒤에 `[ENTER]`를 기록하여 다음 문단의 해설을 시작하고, 해설 시작부터 끝까지 원문이 줄바꿈을 요구하는 위치에는 `[강제줄바꿈]`을 기록한다 → 최종 출력: 변환기는 `[강제줄바꿈]`을 HWP `Ctrl+Shift+Enter`(강제 줄 나눔)로 처리한다 → 예외·검토 필요: 이 미주 전용 `[강제줄바꿈]`은 기존 `[enter]`와 다르다. `[enter]`는 문제 본문 전사의 일반 문단 나눔이고, `[강제줄바꿈]`은 하나의 해설 문단 안에서 새 줄을 보존한다.

```text
1) 정답 3[ENTER]해설[강제줄바꿈]해설[강제줄바꿈]해설끝
```

62. [미주 데이터 출력 형식] 판단 조건: 미주 데이터 구역을 출력하면 → 처리 방법: 문항 번호 오름차순으로 정확히 연결된 데이터만 배열한다 → 최종 출력: 전체 형식은 다음과 같다 → 예외·검토 필요: 각 `<미주 N>`에는 해당 번호의 답지·해설만 둔다.

```text
<미주 N>
정답: ...
해설:
...
</미주 N>
```

63. [빠른답지만 있는 미주] 판단 조건: 빠른답지만 있고 연결된 해설이 없으면 → 처리 방법: `정답:`만 기록한다 → 최종 출력: 다음 형식을 사용한다 → 예외·검토 필요: 해설을 추정하여 추가하지 않는다.

```text
<미주 N>
정답: ③
</미주 N>
```

64. [해설지만 있는 미주] 판단 조건: 해설지만 있고 빠른답지가 없으면 → 처리 방법: `해설:`만 기록한다 → 최종 출력: 다음 형식을 사용한다 → 예외·검토 필요: `정답:`을 추정하거나 추가하지 않는다.

```text
<미주 N>
해설:
...
</미주 N>
```

65. [미주 데이터 없음] 판단 조건: 연결할 답지 또는 해설 원본 데이터가 전혀 없으면 → 처리 방법: 미주 데이터 구역에 개별 `<미주 N>`을 만들지 않는다 → 최종 출력: `미주 데이터 없음`만 출력한다 → 예외·검토 필요: 빈 데이터 블록이나 추정 정답·해설을 추가하지 않는다.
66. [검수와 산출물 폴더] 판단 조건: 최종 전사와 작업 파일을 검수·저장할 때 → 처리 방법: 모든 `#N`과 `<미주 N>`의 정확한 대응, 만들어 낸 정답·해설의 부재, 빠른답지와 해설 결론의 충돌, 모든 `@` 짝과 `^{...}` 지수, `[enter]` 배치를 확인하고, 불확실한 번호·이미지·판독성 메모를 `검수 메모`에 기록한다 → 최종 출력: TXT와 그 TXT가 참조하는 모든 PNG는 하나의 같은 작업 폴더에 저장하고 `[image: 파일명.png]`에는 하위 폴더나 절대 경로 없이 파일명만 쓴다 → 예외·검토 필요: 서로 다른 전사 작업의 TXT·이미지를 같은 작업 폴더에 섞지 않는다.
